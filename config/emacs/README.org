#+title: Sky Emacs
#+startup: overview

* Purpose
This is the clean personal Emacs configuration experiment.

This is the active clean Emacs config. Doom is frozen as reference material
under =~/.config/doom=.

* Ownership
#+begin_src text
Nix owns external tools, language servers, formatters, fonts, services, wrappers, and links.
Emacs owns Emacs packages, editor behavior, keymaps, completion, LSP client behavior, Org, Dired, terminal, and theme faces.
Projects own project-local metadata such as .dir-locals.el and language-server hints.
#+end_src

* Modeline And Theme
Modeline ownership is split deliberately:

#+begin_src text
sk-ui.el          Doom modeline layout and noise knobs
sk-theme.el       Sky face colors for mode-line and doom-modeline faces
#+end_src

The modeline should stay compact and useful:

#+begin_src text
active window     clear Sky surface background, accent bar, readable file/project state
inactive window   dim background, muted text, no heavy border
status colors     success/warning/danger tokens for LSP, diagnostics, and modified state
noise policy      hide encoding/env/workspace/chat counters unless we decide they matter
#+end_src

=solaire-mode= is enabled to give secondary buffers such as help, Magit, Dired,
and other utility views a Sky panel background. It should create visual depth
without changing normal editing buffers.

=spacious-padding-mode= adds a small amount of breathing room around frames,
dividers, fringes, and mode/header lines. The values are intentionally modest so
the editor keeps its working density.

The tab line is enabled as a polished buffer strip. It uses Nerd Icons, a compact
buffer name, and Sky faces that mirror the modeline palette.

Utility buffers use a consistent side-window policy:

#+begin_src text
help / eldoc / apropos       right panel
dired / ibuffer / magit      right panel
eshell / vterm               right panel
warnings / compile output    bottom panel
#+end_src

* Completion
Completion is layered deliberately:

#+begin_src text
minibuffer        Vertico + Orderless + Marginalia + Consult + Embark

code sources      snippet triggers first, then mode/LSP CAPFs and Cape fallbacks
prose sources     Org/text CAPFs first, then Cape file and dabbrev

snippet fields    TAB / S-TAB move next / previous while a snippet is active

docs              Corfu popupinfo shows candidate documentation when the source provides it
#+end_src

Completion keys:

#+begin_src text
C-j / C-k         next / previous completion candidate
C-l               accept completion candidate
C-h               quit completion popup
M-/               start completion at point; Corfu displays the popup
M-d               toggle Corfu documentation popup while completion is open
#+end_src

Documentation availability depends on the completion source. LSP candidates may
provide docs. File names, keywords, dabbrev, snippets, and many local symbols
usually do not. Snippet triggers are offered in Corfu for code buffers and
expand when accepted. Current tracked snippets cover QML/Quickshell.

* Tree-Sitter Policy
Nix owns Tree-sitter parser delivery for clean Emacs. Emacs reads the parser
bundle from =SK_EMACS_TREE_SITTER_GRAMMAR_PATH= and only remaps modes when both
the grammar and built-in TS major mode are available.

=font-lock-maximum-decoration= is set to =t= so modes use their richest
available highlighting level. This only affects faces that the major mode
actually marks; it does not invent semantic highlighting for underpowered modes.

Routed to built-in TS modes:

#+begin_src text
python        python-ts-mode
rust          rust-ts-mode
c / c++       c-ts-mode / c++-ts-mode
shell         bash-ts-mode
lua           lua-ts-mode
nix           nix-ts-mode
haskell       haskell-ts-mode
markdown      markdown-ts-mode
json          json-ts-mode
yaml          yaml-ts-mode
css           css-ts-mode
javascript    js-ts-mode
typescript    typescript-ts-mode / tsx-ts-mode
toml          toml-ts-mode
html          html-ts-mode
#+end_src

Grammar available, classic mode retained until we deliberately choose an Emacs
TS mode package or confirm the built-in mode exists:

#+begin_src text
glsl
org
#+end_src

QML uses =sk-qml-ts-mode=, a small local mode backed by the =qmljs=
Tree-sitter parser. It keeps =qml-mode= as the fallback when the parser is not
available and avoids ad hoc regex highlighting.

The local QML mode exists because clean Emacs does not currently provide a
built-in QML TS major mode that covers our Quickshell files well. It owns only
editor parsing/highlighting policy. Runtime QML structure still belongs to the
Quickshell tree, its =qmldir= files, and the global project guidelines.

Current QML coverage:

#+begin_src text
imports             import modules and import paths
object types         Canvas, PanelWindow, RowLayout, local components
properties           QML property declarations and bindings
attached props       WlrLayershell.layer, Layout.fillWidth
JS blocks            var declarations, initializer uses, binary expressions,
                     function calls, and function argument identifiers
constants            true, false, null
strings/comments     parser-backed string and comment nodes
#+end_src

QML LSP support stays intentionally modest. =qmlls-wrapped= supplies Qt and
Quickshell import paths, completion, Eldoc, Flymake, and formatting support,
but hover prose and jump precision depend on Qt/Quickshell metadata and
=qmlls= behavior. Sparse hover text is not, by itself, an editor wiring failure.

QML validation must stay light. Repeated automated completion probes can spawn
many =qmlls= children and wedge the Emacs daemon. Prefer manual checks from a
real frame. Safe scripted checks should use temp buffers with mode hooks
disabled and verify only parser/font-lock behavior.

Reference material:

#+begin_src text
xhcoding/qml-ts-mode  https://github.com/xhcoding/qml-ts-mode
#+end_src

Use external QML modes as references, not automatic replacements. The
=xhcoding/qml-ts-mode= package is useful for indentation, comment setup,
imenu/defun navigation, operators, brackets, delimiters, escape sequences, and
QML keyword coverage. Our local mode stays the owner until a borrowed piece is
tested against real Quickshell files and proves better.

NVF is not edited by this policy. It remains a reference and already owns its
own Tree-sitter setup through NVF.

* LSP Policy
Eglot is the clean Emacs LSP client. Servers are started for real file buffers
only, and =eglot-autoshutdown= stays disabled so project language servers are
not repeatedly started and killed during normal buffer churn.
